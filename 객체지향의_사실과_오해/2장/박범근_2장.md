# 이상한 나라의 객체
> 객체지향 패러다임은 지식을 추상화하고 추상화한 지식을 객체 안에 캡슐화함으로써 실세계 문제에 내재된 복잡성을 관리하려고 한다.

## 객체지향과 인지 능력
- 세상을 자율적이고 독립적인 객체들로 분해할 수 있는 인간의 기본적인 이지 능력에 기반을 두고 있다.
- 객체란, 인간이 분명하게 인지하고 구별할 수 있는 물리적인 또는 개념적인 경계에 지닌 어떤 것!
  - 더 작은 문제로 쪼개자!
- 하지만, 객체지향의 패러다임은 현실 세계 모방이 아닌 <b>새로운 세계를 창조</b>하는 것.

### 앨리스 객체
- 특정 시점의 앨리스의 상태란 특정 시점에서의 앨리스의 키를 의미.
- 앨리스의 상태를 결정하는 것은 행동이지만, 행동의 결과를 결정하는 것은 상태
  - 이것은 행동 간의 순서가 중요하다는 것을 의미
  - 문을 통과하는 행동이 성공하려면? 이전에 음료나 케이크를 먹어서 키(상태)를 줄여야 한다는 것

## 객체, 그리고 소프트웨어 나라
- 객체의 다양한 특성을 효과적으로 설명하기 위해서는 객체를 상태(state), 행동(behavior), 식별자(identity)를 지닌 실체로 보는 것이 가장 효과적

### 상태
- 모든 객체의 상태는 단순한 값과 객체의 조합으로 표현이 가능.
- 프로퍼티(property) : 객체의 상태를 구성하는 모든 특징을 아우르는 개념 (정적)
- 프로퍼티 값(property value) : 시간이 흐름에 따라 변경 (동적)
- 링크(link) : 객체와 객체 사이에 링크가 존재해서 요청을 주고 받는 것 (메시지를 주고 받는 행위)

### 행동
- 객체의 행동은 상태에 영향을 받음
- 객체의 행동은 상태를 변경시킴
- 외부의 요청 또는 수신된 메시지에 응답하기 위해 동작하고 반응하는 활동
  - 행동의 결과로 객체는 자신의 상태를 변경 or 다른 객체에게 메시지를 전달이 가능.

## 행동이 상태를 결정한다.
> 상태를 먼저 결정하고, 행동을 결정하지 말자!
- 객체지향 설계는 어플리케이션에 필요한 협력을 생각하고 협력에 참여하는 데 필요한 행동을 생각한 후 행동을 수행할 객체를 선택하는 방식으로 수행.
- <b>객체의 행동을 먼저 결정하고 그 후에 행동에 적절한 상태</b>를 선택하자.
- 협력 안에서 객체의 행동 => 협력에 참여하면서 완수해야 하는 책임

## 은유와 객체
- 추상화(abstraction) : 실제의 사물에서 자신이 원하는 특성만 취하고 필요 없는 부분을 추려 핵심만 포현하는 행위
- 하지만, 모방과 추상화란 개념만으로는 현실과 소프트웨어 사이의 관계를 깔끔하게 설명하지 못함.

### 의인화
- 수동적인 현실 세계의 객체와 달리, SW 세계의 객체는 능동적이다.
- 현실 객체보다 더 많은 일을 할 수 있는 SW 객체의 특징을 의인화(anthropomorphism)이라고 부른다.
- 객체지향 세계의 객체는 현실보다 더 많은 특징과 능력을 보유한 객체들로 넘쳐난다!

### 은유
- 하나의 의미를 다른 것을 이용해 전달한다는 의미
- 현실 세계인 도메인에서 사용되는 이름을 객체에게 부여하자!
- SW에서 사람들이 생각하는 모습을 실제 객체 이름을 사용함으로써 표현적 차이를 줄여 쉽게 예측하게끔!

  > 창조한 객체의 특성을 상기시킬 수 있다면 현실 속의 객체의 이름을 이용해 객체를 묘사해라.
